## 1. Что тебе понадобится

Перед началом убедись, что у тебя есть:

- Компьютер с **Windows 10/11** (для сборки .exe обязательно Windows)
- Доступ в интернет
- Аккаунт на [GitHub](https://github.com) (бесплатный)

Нужно установить:
- **Node.js** (движок для запуска JavaScript)
- **Git** (для работы с кодом и GitHub)

---

## 2. Установка Node.js

1. Зайди на сайт [https://nodejs.org](https://nodejs.org)
2. Нажми большую зелёную кнопку **"LTS"** (это стабильная версия)
3. Скачается файл типа `node-v20.x.x-x64.msi`
4. Запусти его и нажимай **Next** на всех шагах (всё оставь по умолчанию)
5. На шаге **"Tools for Native Modules"** — поставь галочку если есть, жми **Next**
6. Нажми **Install**, дождись завершения

**Проверка:** Открой **Командную строку** (Win+R → `cmd` → Enter) и введи:
```
node --version
```
Должно написать что-то вроде `v20.15.0`. Если написало — Node.js установлен.

---

## 3. Установка Git

1. Зайди на сайт [https://git-scm.com/download/win](https://git-scm.com/download/win)
2. Скачается установщик автоматически
3. Запусти и нажимай **Next** на всех шагах (всё по умолчанию)
4. На шаге **"Adjusting your PATH environment"** выбери **"Git from the command line and also from 3rd-party software"**
5. Дожди установки

**Проверка:** В командной строке введи:
```
git --version
```
Должно написать `git version 2.xx.x`

## 4. Загрузить код на GitHub

### Шаг 4.1 — Создать репозиторий на GitHub

1. Зайди на [https://github.com](https://github.com) и войди в аккаунт
2. Нажми кнопку **"+"** в правом верхнем углу → **"New repository"**
3. Заполни:
   - **Repository name:** `omni-client` (или любое другое название)
   - **Description:** опционально
   - Выбери **Private** (чтобы код был только у тебя) или **Public**
   - **НЕ ставь** галочку на "Add a README file"
4. Нажми **"Create repository"**
5. Запомни или скопируй ссылку на репозиторий, она будет вида:  
   `https://github.com/твой_логин/omni-client.git`

### Шаг 4.2 — Скопировать папку omni-client с Replit

1. В Replit скачай файлы проекта:
   - Нажми на три точки рядом с папкой `omni-client` в файловом дереве
   - Выбери **Download** (скачает .zip архив)
2. Распакуй архив на компьютере, например в `C:\Projects\omni-client\`

### Шаг 4.3 — Загрузить код на GitHub

Открой **командную строку** и введи команды по одной (после каждой нажимай Enter):

```bash
cd C:\Projects\omni-client
```
*(заменить путь на тот, куда распаковал)*

```bash
git init
```

```bash
git add .
```

```bash
git commit -m "Initial commit"
```

```bash
git branch -M main
```

```bash
git remote add origin https://github.com/ТВОЙ_ЛОГИН/omni-client.git
```
*(заменить ТВОЙ_ЛОГИН на свой логин GitHub)*

```bash
git push -u origin main
```

Если Git попросит логин/пароль GitHub — введи их.  
> Если не принимает пароль — нужен **Personal Access Token**. Читай: [github.com/settings/tokens](https://github.com/settings/tokens) → Generate new token (classic) → ставь галочку **repo** → Copy token → используй вместо пароля.

После этого обнови страницу репозитория на GitHub — код должен появиться.

## 5. Запустить приложение локально

> Это нужно если хочешь запустить приложение прямо на своём компьютере без сборки в .exe, например для тестирования.

### Шаг 5.1 — Установить зависимости

Открой командную строку в папке `omni-client`:

```bash
cd C:\Projects\omni-client
npm install
```

Это скачает все необходимые библиотеки (~500 МБ). Подождёт 2-5 минут.

### Шаг 5.2 — Запустить в режиме разработки

```bash
npm run dev
```

Должно открыться окно приложения Omni.

> Если окно не открылось — жди 10-15 секунд, иногда нужно время на запуск.  
> Для выхода нажми **Ctrl+C** в командной строке.

## 6. Собрать .exe вручную

> Этот способ — собрать .exe прямо на своём Windows компьютере. Самый быстрый способ получить готовый файл.

### Требования:
- Windows 10/11
- Node.js установлен (шаг 2)
- Зависимости установлены (`npm install`, шаг 5.1)

### Сборка:

```bash
cd C:\Projects\omni-client
npm run build:win
```

Это займёт **3-10 минут**. По завершению в папке `release\` появятся файлы:

```
release\
  Omni Setup 3.0.0.exe    ← установщик (рекомендуется)
  Omni 3.0.0.exe           ← portable версия (запускается без установки)
```

**Omni Setup 3.0.0.exe** — обычный установщик. После запуска устанавливает Omni в `C:\Program Files\Omni` и создаёт ярлык на рабочем столе.

**Omni 3.0.0.exe** — portable, можно запускать с флешки, не требует установки.

## 7. Собрать .exe через GitHub Actions

> GitHub Actions — это бесплатный облачный сервис, который автоматически собирает .exe при каждом обновлении кода. Очень удобно — не нужен Windows.

### Как это работает:
1. Ты загружаешь код на GitHub
2. GitHub автоматически запускает сборку на своём Windows-сервере
3. Через 5-10 минут готовый .exe можно скачать

### Шаг 7.1 — Файл сборки уже создан

Файл `.github/workflows/build.yml` уже есть в проекте. Он запускает сборку автоматически.

### Шаг 7.2 — Проверить что Actions включены

1. Зайди на страницу репозитория GitHub
2. Кликни вкладку **Actions** (вверху страницы)
3. Если видишь кнопку **"I understand my workflows, go ahead and enable them"** — нажми её

### Шаг 7.3 — Запустить сборку

Сборка запускается **автоматически** при любом push в `main`. После шага 4.3 она уже должна была запуститься.

Чтобы запустить вручную:
1. Зайди на **Actions** в репозитории
2. Слева выбери **"Build Omni Client"**
3. Нажми **"Run workflow"** → **"Run workflow"**

### Шаг 7.4 — Следить за процессом

1. На вкладке **Actions** появится запись с жёлтым кружком (выполняется) или зелёной галочкой (готово)
2. Кликни на неё чтобы видеть прогресс
3. Обычно сборка занимает **5-10 минут**

## 8. Как скачать готовый .exe из GitHub

1. Зайди на **Actions** → кликни на завершённую сборку (зелёная галочка ✅)
2. Прокрути вниз до раздела **"Artifacts"**
3. Увидишь файлы:
   - `OmniClient-Setup-win-x64` — установщик
   - `OmniClient-Portable-win-x64` — portable версия
4. Нажми на нужный — скачается .zip архив
5. Распакуй архив — внутри будет `.exe` файл

> Артефакты хранятся **30 дней**. Для постоянного хранения — сделай релиз (шаг 9).

## 9. Создать публичный релиз

> Релиз — это публичная страница с готовым .exe файлом, который можно скачать в любое время. Как у обычных программ.

### Шаг 9.1 — Создать тег версии

В командной строке в папке проекта:

```bash
git tag v1.0.0
git push origin v1.0.0
```

Это создаст версию `v1.0.0`.

### Шаг 9.2 — GitHub Actions автоматически создаст релиз

После push тега Actions запустится ещё раз и автоматически:
1. Соберёт .exe
2. Создаст страницу **Releases** с файлами для скачивания

### Шаг 9.3 — Найти релиз

На странице репозитория справа есть раздел **"Releases"** — кликни на него.  
Там будет кнопка **"Assets"** с файлами для скачивания.

### Следующие версии

Чтобы выпустить версию 1.0.1 после изменений:

```bash
git add .
git commit -m "Описание изменений"
git push origin main
git tag v1.0.1
git push origin v1.0.1
```

## 11. Частые ошибки

### ❌ `node: command not found` или `'node' is not recognized`
**Решение:** Node.js не установлен или не добавлен в PATH.  
Переустанови Node.js с [nodejs.org](https://nodejs.org), обязательно оставь галочку "Add to PATH".  
После установки перезапусти командную строку.

---

### ❌ `npm install` зависло или долго идёт
**Решение:** Это нормально, может занять 5 минут. Просто жди.  
Если зависло больше 15 минут — нажми Ctrl+C и попробуй снова.

---

### ❌ `Error: Cannot find module 'electron'`
**Решение:** Зависимости не установлены. Запусти:
```bash
npm install
```

---

### ❌ При сборке: `error: ENOENT: no such file or directory, open 'src/renderer/assets/icon.ico'`
**Решение:** Нет иконки. Создай файл-заглушку:
1. Скачай любой .ico файл из интернета
2. Положи его в `src/renderer/assets/icon.ico`

Или убери иконку из `package.json` — найди строки:
```json
"icon": "src/renderer/assets/icon.ico"
```
И удали их (3 штуки в секции `win` и `nsis`).

---

### ❌ GitHub Actions: красный крестик ❌ на сборке
**Решение:**
1. Кликни на красный крестик
2. Кликни на упавший шаг (например "Build Electron app")
3. Прочитай текст ошибки
4. Самые частые причины:
   - Нет иконки (см. выше)
   - Синтаксическая ошибка в коде

---

### ❌ Windows пишет "Неизвестный издатель" при запуске .exe
**Решение:** Это нормально для неподписанных приложений.  
Нажми **"Подробнее"** → **"Выполнить в любом случае"**.  
Для избавления от этого предупреждения нужен платный сертификат подписи кода.

---

### ❌ Приложение открывается, но не подключается к серверу
**Решение:** Сервер `omnii.duckdns.org:3000` недоступен или выключен.  
Проверь что сервер (`server.js`) запущен и доступен.

## Смена сервера

Если адрес сервера изменился, отредактируй файл `src/renderer/lib/api.js`:

```javascript
// Строка 1: поменяй адрес сервера
const BASE_URL = 'https://НОВЫЙ_АДРЕС:3000'
```

И файл `src/renderer/lib/ws.js`:

```javascript
// Строка 1: поменяй адрес WebSocket
const WS_URL = 'wss://НОВЫЙ_АДРЕС:3000/ws?token={token}&version=2.0.0'
```
